# 功能矩阵

## 用户画像

| 画像 | 描述 | 窗口大小 | 使用模式 |
|------|------|---------|---------|
| **重度社交** | 500+ 好友，活跃在多个群组，追踪所有人 | 大（1080p+） | 一直开着应用，持续监控动态 |
| **普通桌面** | 50-200 好友，VR 前后查看 | 中大 | 偶尔打开，快速看看好友状态 |
| **小窗口** | VRCX 作为侧边栏配合其他应用 | 小（400-600px） | 需要一目了然的紧凑信息 |
| **VR 用户** | 游戏中使用手腕覆盖层 | 手腕覆盖层 | 最小交互，只能瞥一眼 |

## 如何使用这个矩阵

1. **添加功能前**：在矩阵中找到对应行。检查哪些画像关心这个功能。
2. **如果某画像是 ⛔**：不要强塞——要么跳过该画像，要么提供降级版本。
3. **如果小窗口是 ✅**：该功能**必须**在 ~400px 宽度下可用。先设计紧凑版。
4. **如果 VR 是 ✅**：该功能必须支持最小交互（只能点击/瞥一眼）。
5. **决策后更新**：在决策记录列加日期 + 备注，这样你记得**为什么**这么选。
